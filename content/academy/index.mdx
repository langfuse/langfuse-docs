# Langfuse Academy

Langfuse Academy teaches the operating model behind LLM observability. The goal is not to walk through product screens or individual features. The goal is to help you reason about how teams capture evidence, learn from production behavior, test changes, and decide what to ship.

## What this academy teaches

Traditional observability is still necessary for AI systems, but it is not enough. An LLM application can be technically healthy and still fail the user: the answer can be wrong, off-brand, unsafe, incomplete, or simply unhelpful. That is why LLM observability sits closer to product quality and iteration than classic infrastructure monitoring usually does.

This section teaches a repeatable loop for working on that problem. You will learn how to:

- Capture useful traces
- Monitor live production behavior
- Turn recurring cases into datasets
- Run controlled experiments
- Evaluate whether a change is actually better

## Who this is for

This academy is for practitioners building or operating LLM features, assistants, or agents in production. It is especially useful if you already have some telemetry and are now asking questions like:

- How do we know what the system actually did?
- How do we tell noise from a real quality problem?
- Which cases should become repeatable tests?
- How do we compare changes without fooling ourselves?
- When is a result strong enough to ship?

If you are looking for implementation steps, SDK details, or product workflows, treat the academy as the conceptual layer and the main docs as the how-to layer.

## The loop at a glance

![The AI engineering loop](/images/academy/loop-overview.png)

The academy is organized around a five-step loop:

1. [Trace](/academy/tracing) what happened.
2. [Monitor](/academy/monitoring) live behavior and prioritize what matters.
3. [Build datasets](/academy/datasets) from recurring cases you want to test repeatedly.
4. [Experiment](/academy/experiments) by changing one variable at a time.
5. [Evaluate](/academy/evaluate) whether the result is good enough to ship.

## How to use this section

Start with [The LLM Observability Loop](/academy/loop) if you want the full mental model first. Then read the loop step that matches the problem you are working on right now.

The core loop pages focus on decisions, artifacts, and sequencing. The methods pages cover reusable techniques such as manual review, code-based evaluators, and LLM-as-a-judge.

You do not need to implement everything at once. But the pieces reinforce each other. Weak traces make monitoring vague. Weak datasets make experiments misleading. Weak evaluation makes shipping decisions political instead of empirical.

The academy exists to make that loop explicit.

Langfuse Cloud is now live in a dedicated Japan region, hosted in Tokyo (`ap-northeast-1`). Traces, prompts, and evaluation data stay in Japan, and trace ingestion latency for applications running in APAC drops to Tokyo instead of Dublin or Oregon.

This is the right region to pick when you need to:

- Keep traces, prompts, and evaluation data inside Japan for data residency or regulatory reasons
- Cut trace ingestion latency for applications running in Japan or broader APAC
- Run on managed Langfuse Cloud without having to self-host

This is the full Langfuse platform - observability, prompt management, evaluations, datasets, and metrics. Nothing is cut down compared to Cloud EU or Cloud US.

## What's included

- **Full platform parity** with Cloud EU and Cloud US — same features, same API surface, same SDK support
- **Data stays in Japan** — traces, prompts, and evaluation records are stored and processed in Tokyo
- **Japanese-speaking team** based in Tokyo for sales and ongoing support
- **Migration path** from Cloud EU or Cloud US — [data migration script](/guides/cookbook/example_data_migration). Reach out to use if you have a very large amount of trace data to migrate (>1M observations)

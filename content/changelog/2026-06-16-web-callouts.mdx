You can now configure a Web Callout for a project and trigger it directly from trace, observation, and session views. Langfuse sends a synchronous backend `POST` request with the project ID plus the selected trace, observation, and session IDs, so your service can fetch the full context through the API and open the right workflow.

Use it to open an internal support case from a trace, send a failing observation to an incident workflow, or jump from a session to a custom investigation tool. Project admins configure the endpoint, toast text, enablement, and static request headers under **Project Settings** > **Integrations** > **Web Callouts**.

Callouts send identifiers only, support encrypted static headers for backend authentication, validate targets before each request, follow redirects safely, time out after 5 seconds, and do not retry. They are manually triggered from the UI rather than event-driven webhooks.

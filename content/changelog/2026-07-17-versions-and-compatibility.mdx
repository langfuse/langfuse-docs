You can now look up how Langfuse server, SDK, and API versions relate on a single page: [Versions & Compatibility](/docs/compatibility). It covers the SDK ↔ server compatibility matrix with minimum server versions, API version availability across Langfuse Cloud and self-hosted deployments, and answers to the most common compatibility questions.

This is useful when you are planning an SDK upgrade on a self-hosted deployment (check the minimum server version first), deciding when to move from the v1 to the v2 Observations and Metrics APIs, or verifying which SDK versions deliver real-time data on the [Langfuse v4](/docs/v4) data model.

Alongside it, the new [deprecations page](/self-hosting/upgrade/deprecations) lists every deprecated API, ingestion path, and integration with its replacement, the removal date on Langfuse Cloud (November 30, 2026 for the v4 transition), and the server version that removes it on self-hosted deployments.

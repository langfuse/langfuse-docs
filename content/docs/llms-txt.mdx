# llms.txt example: the Langfuse docs index for LLMs

`llms.txt` is a proposed standard, published at [llmstxt.org](https://llmstxt.org/), for a markdown file served at the root of a website that gives LLMs a concise, curated index of the site's content. Langfuse publishes one at [langfuse.com/llms.txt](https://langfuse.com/llms.txt). This page explains the format using our own file as a working example, and shows how it is generated and consumed.

## What llms.txt is [#what-is-llms-txt]

An `llms.txt` file is a plain markdown file at the `/llms.txt` path of a website. The proposal describes a simple structure: an H1 with the site name, a blockquote that summarizes the site in one paragraph, and sections containing lists of links to markdown files with more detail.

The format exists because raw HTML pages are a poor fit for LLM context windows: navigation, scripts, and boilerplate crowd out the actual content, and an agent has no cheap way to discover which pages matter. A curated markdown index lets an agent read one small file, decide which page it needs, and fetch that page directly.

## The Langfuse llms.txt, annotated [#langfuse-llms-txt-example]

The Langfuse `llms.txt` is a concise index: a one-paragraph description of Langfuse, followed by sections that point agents at richer resources (the docs MCP server and the Langfuse skill) and per-section link files. As of July 2026, an abridged excerpt looks like this:

```md
# Langfuse

> Langfuse is an **open-source AI engineering platform** ([GitHub](https://github.com/langfuse/langfuse)) that helps teams collaboratively debug, analyze, and iterate on their LLM applications. All platform features are natively integrated to accelerate the development workflow.

## Langfuse Docs MCP Server

Connect to the Langfuse Docs MCP server to access documentation directly in your AI editor:

- **Endpoint**: `https://langfuse.com/api/mcp`
- **Transport**: `streamableHttp`
- **Documentation**: [Langfuse Docs MCP Server](https://langfuse.com/docs/docs-mcp)

## Langfuse Skill for AI Coding Agents

Install the Langfuse skill before implementing anything with Langfuse. ...

## Tracing Best Practices

Before instrumenting an application with Langfuse, read [What does a good trace look like?](https://langfuse.com/docs/observability/best-practices.md) ...

## Docs

For the full list with links to each page, see: https://langfuse.com/llms-docs.txt

Pages: Overview, ..., Ask AI, ..., Docs MCP Server, ...
```

A few deliberate choices in this structure:

- The H1 and blockquote follow the llmstxt.org proposal: they name the site and summarize it in one quotable paragraph.
- The sections that follow tell agents where richer context lives (the [docs MCP server](/docs/docs-mcp) and the [Langfuse skill](https://github.com/langfuse/skills/tree/main/skills/langfuse)) before any link list, because an agent that connects to those sources gets search instead of a static index.
- Each content section ("Docs", "Integrations", "Optional: Self-Hosting") stays short in the main file: it lists page titles and links to a dedicated sub-file, so the main file remains small enough to paste into any context window.

## Sub-files keep the main file small [#sub-files]

The main file links to three sub-files, one per content section: [llms-docs.txt](https://langfuse.com/llms-docs.txt), [llms-integrations.txt](https://langfuse.com/llms-integrations.txt), and [llms-self-hosting.txt](https://langfuse.com/llms-self-hosting.txt). Each sub-file lists every page in that section as a markdown link with the page's real frontmatter title and description:

```md
# Langfuse Docs

- [Ask AI](https://langfuse.com/docs/ask-ai.md): Ask AI questions about Langfuse features, integrations, SDK usage, and best practices. ...
```

Every link points at the `.md` version of the page. Langfuse serves each docs page as plain markdown when you append `.md` to its URL (for example, [langfuse.com/docs/docs-mcp.md](https://langfuse.com/docs/docs-mcp.md)), so an agent following a sub-file link receives clean markdown rather than HTML.

## How the file is generated [#how-llms-txt-is-generated]

The file is generated at build time by the open-source [generate_llms_txt.js](https://github.com/langfuse/langfuse-docs/blob/main/scripts/generate_llms_txt.js) script, not maintained by hand. The build works in three steps:

1. The build produces the sitemap plus a per-page metadata file containing each page's frontmatter `title` and `description`.
2. The script reads every URL from the sitemap and groups pages by their top-level path (`docs`, `integrations`, `self-hosting`).
3. It writes one sub-file per section and a concise main `llms.txt` that links to them.

Because titles and descriptions come straight from page frontmatter, the file never drifts from the docs: publishing or renaming a page updates `llms.txt` on the next deploy. We first shipped the file in November 2024 ([changelog](/changelog/2024-11-17-llms-txt)) and have extended it since.

## How agents and users consume it [#how-to-use-llms-txt]

The simplest use is to give your AI editor or agent the URL `https://langfuse.com/llms.txt` as context; the agent can then follow the sub-file links and fetch individual pages as markdown. For deeper integration, the docs MCP server is the richer alternative.

| Method                            | Best for                                 | How                                                        |
| --------------------------------- | ---------------------------------------- | ---------------------------------------------------------- |
| `llms.txt`                        | One-shot context in any LLM tool         | Paste `https://langfuse.com/llms.txt`                      |
| `.md` page URLs                   | Fetching a single page as clean markdown | Append `.md` to any docs URL                               |
| [Docs MCP server](/docs/docs-mcp) | Agents that support MCP                  | Endpoint `https://langfuse.com/api/mcp` (`streamableHttp`) |
| [Ask AI](/docs/ask-ai)            | Interactive questions in the browser     | Chat on langfuse.com                                       |

The MCP server exposes search over the Langfuse documentation, GitHub issues, and discussions, so an agent can query for exactly the page it needs instead of scanning an index.

## FAQ [#faq]

### Is llms.txt an official web standard?

No. It is a community proposal published at [llmstxt.org](https://llmstxt.org/) rather than a specification ratified by a standards body. Many documentation sites, including Langfuse, have adopted it voluntarily because the format is simple and useful today.

### How do I create an llms.txt file for my own docs site?

Generate it from metadata you already maintain instead of writing it by hand. Langfuse builds its file from the sitemap and each page's frontmatter title and description, using a [script](https://github.com/langfuse/langfuse-docs/blob/main/scripts/generate_llms_txt.js) that runs after every build, so the index stays synchronized with the docs automatically. Any static site generator that exposes a sitemap and page metadata can follow the same pattern.

### Do AI tools automatically fetch llms.txt?

Support varies by tool, and as of July 2026 there is no guarantee that a crawler or agent fetches the file on its own. Most value today comes from explicitly pointing a tool at the URL, for example by adding it as a docs source in an AI code editor or passing it to an agent as context.

# How to Answer Support Questions

This guide outlines how to handle common support questions in a consistent way. Use each question as a playbook: identify the issue, check the relevant systems, and reply with the next clear action for the user.

## I have higher costs than usual / I was charged unexpectedly

1. Go to Stripe.
2. Search for the user's account via domain/email and review the latest invoice, subscription, and billing history.
3. Check whether the increase came from a plan change, usage increase, additional seats, or a one-off charge.
4. Cross-check the account in Langfuse Impersonation View to confirm workspace usage and any recent changes that could explain the increase.
5. Reply with the specific reason for the higher cost and link the relevant invoice or billing page if helpful.
6. If the charge still looks wrong, escalate internally before confirming any refund or billing correction.
7. __Important__ to note is that sometimes Langfuse previously logged unrelated events from OTEL which led to higher costs. [Read more](https://langfuse.com/faq/all/existing-otel-setup#unwanted-spans-in-langfuse:~:text=Your%20Langfuse%20dashboard,libraries%20if%20needed)
8. For refunds above 2K loop in the COO on Pylon as @tag

## Do you have a bug bounty program / Security Request

a. If the person mentions a real security thread (as shown [here](https://langfuse.com/security/responsible-disclosure#bug-bounty-program)), instantly escalate to a member of the engineering team!
b. If the person offers his services or minor feedback/recommendations, use the following message:
Hello X,
thank you very much for your feedback. At the current time, Langfuse doesn't offer a formal bug bounty program.
Please have a look at [at this page](https://langfuse.com/security/responsible-disclosure#bug-bounty-program) as it offers more information in this matter. 
I hope I was able to help you with your enquire.
Best,
Y


## I am unable to see view X in langfuse cloud

1. First verify that Fast Mode (Preview) is toggled on. 
Hello X,
I am Y, a support engineer at Langfuse. Have you ensured that Fast Mode on the left sidebar is toggled on?
Please let me know if this solves your issue?
Best,
Caleb
2. If the response shows that they hve toggled the Preview on but the issue remains, try to find the root of the problem by yourself and correspond with another support engineer. 
3. If the issue pertains, escalate to engineering and thank the customer for their input.
Hello X,
the support team has talked about this issue internally and has escalated the ticket to engineering to provide a fix.
Thank you very much for your feedback.
Best,
Y

## Our ingestion is very slow for Z, is this an issue on our side? 
1. First check our status page status.langfuse.com 
1.1 If there is a known issue with Z, answer in the following way:
Hello X,
we are aware of this issue and are working on it.
We will provide an update on our status page as soon as we have it fixed.
Best,
You
2. If this issue is not known, make sure that this is not a common issue by comparing to other support tickets
2.1 If other support tickets experience the same, escalate to engineering
3. If there are no other support tickets referencing this issue, keep the message in mind in case it is the first message of many
4. Ask the customer about their setup. Are they self hosted or running on cloud
4.1 If they are self hosted, ask them about their setup and which version of langfuse they are using
4.2 If they are running on cloud, act in the following way
4.2.1 Hobby: Notify them to open a github discussion on this matter and redirect them to relevant pages
4.2.2 Core: If the issue seems like a minor langfuse implementation bug (if they provided code), escalate to engineer after communication with a support peer and provide a workaround
4.2.3 Pro & Enterprise: Delegate the matter to a member of the engineering team

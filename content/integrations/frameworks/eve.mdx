# Observability for eve with Langfuse

This guide shows you how to integrate **Langfuse** with **eve** to trace and debug your agents. eve emits OpenTelemetry spans for every run, so connecting Langfuse is a matter of pointing those spans at your Langfuse project.

> **What is eve?** [eve](https://eve.dev) is an open-source TypeScript framework from Vercel for building durable agents as ordinary files in a project. It ships with durable execution, a sandbox, tools, subagents, schedules, and built-in OpenTelemetry tracing.

> **What is Langfuse?** [Langfuse](https://langfuse.com) is an open-source observability platform for AI agents and LLM applications. It helps you visualize and monitor model calls, tool usage, cost, latency, and more.

> **How do they work together?** [Every eve run produces a trace](https://eve.dev/docs/guides/instrumentation), and the spans are standard OpenTelemetry, built on the [Vercel AI SDK](/integrations/frameworks/vercel-ai-sdk) telemetry conventions. eve registers its tracer in `agent/instrumentation.ts` via `registerOTel`. By adding the `LangfuseSpanProcessor`, every model call, tool call, and sandbox command flows into Langfuse — where Langfuse maps them to traces, generations, and tool observations.

## Get Started

<Steps>

### Create an eve agent

If you don't have an eve agent yet, scaffold one with the eve CLI:

```bash
npx eve@latest init my-agent
cd my-agent
```

See the full [eve quickstart](https://eve.dev/docs/getting-started) for details.

### Install the Langfuse OpenTelemetry packages

Add the Langfuse span processor and the Vercel OpenTelemetry helper that eve's instrumentation uses:

```bash
npm install @langfuse/otel @vercel/otel
```

### Add your Langfuse credentials

Get your API keys from your [Langfuse project settings](https://cloud.langfuse.com) and add them to your agent's `.env` file. You can sign up for [Langfuse Cloud](https://langfuse.com/cloud) or [self-host Langfuse](https://langfuse.com/self-hosting).

```bash filename=".env"
LANGFUSE_PUBLIC_KEY="pk-lf-..."
LANGFUSE_SECRET_KEY="sk-lf-..."
LANGFUSE_BASE_URL="https://cloud.langfuse.com" # 🇪🇺 EU region
# LANGFUSE_BASE_URL="https://us.cloud.langfuse.com" # 🇺🇸 US region
```

### Register the Langfuse span processor

eve configures telemetry in `agent/instrumentation.ts`. Register the `LangfuseSpanProcessor` on the `registerOTel` call so eve's spans are exported to Langfuse:

```typescript filename="agent/instrumentation.ts"
import { defineInstrumentation } from "eve/instrumentation";
import { registerOTel } from "@vercel/otel";
import { LangfuseSpanProcessor } from "@langfuse/otel";

export default defineInstrumentation({
  setup: ({ agentName }) =>
    registerOTel({
      serviceName: agentName,
      spanProcessors: [
        new LangfuseSpanProcessor({
          // Export only eve's GenAI spans — model calls, tool calls, and agent
          // invocations. eve's durable-workflow orchestration spans and the
          // @vercel/otel HTTP spans are infrastructure noise we keep out of
          // Langfuse.
          shouldExportSpan: ({ otelSpan }) =>
            otelSpan.instrumentationScope.name === "gen_ai",
        }),
      ],
    }),
});
```

The `LangfuseSpanProcessor` reads `LANGFUSE_PUBLIC_KEY`, `LANGFUSE_SECRET_KEY`, and `LANGFUSE_BASE_URL` from the environment.

<Callout type="info">
  Registering the `LangfuseSpanProcessor` via `registerOTel` requires
  `@vercel/otel` v2 or later, which is built on the same OpenTelemetry JS SDK v2
  as `@langfuse/otel` ([vercel/otel#154](https://github.com/vercel/otel/issues/154)).
  The install command above already pulls in a compatible version. If you wire
  Langfuse into a custom OpenTelemetry setup instead, you can register the
  processor on a [`NodeSDK`](/docs/observability/sdk/overview) as well.
</Callout>

<Callout type="info">
  eve runs every turn as a durable workflow, so it emits many low-level
  orchestration spans (instrumentation scope `workflow`) alongside the model and
  tool calls (scope `gen_ai`). The `shouldExportSpan` filter above keeps only the
  `gen_ai` spans, so Langfuse shows the agent invocations, generations, and tool
  calls without the workflow plumbing. Because the dropped spans include the
  run's parent spans, the kept observations can appear as
  [orphaned spans](/docs/observability/sdk/advanced-features#filtering-by-instrumentation-scope)
  in the UI. To keep eve's full run tree instead, drop only the HTTP noise with
  `({ otelSpan }) => otelSpan.instrumentationScope.name !== "@vercel/otel/fetch"`,
  or use `() => true` to export everything. See
  [filtering by instrumentation scope](/docs/observability/sdk/advanced-features#filtering-by-instrumentation-scope)
  for more.
</Callout>

### Define a simple agent

A minimal eve agent is a tool plus a system prompt. The weather agent below — a single `getWeather` tool and a short instruction — produces the kind of run shown in the example trace below:

```ts filename="agent/tools/getWeather.ts"
import { defineTool } from "eve/tools";
import { z } from "zod";

export default defineTool({
  description: "Return mock weather data for a city.",
  inputSchema: z.object({ city: z.string().min(1) }),
  async execute({ city }) {
    return { city, condition: "cloudy", temperatureC: 18 };
  },
});
```

```ts filename="agent/agent.ts"
import { defineAgent } from "eve";

export default defineAgent({
  model: "openai/gpt-4o-mini",
});
```

```markdown filename="agent/instructions.md"
You are a concise weather assistant. When asked about the weather, call the
getWeather tool, then answer in one sentence. Mention that the data is mocked.
```

eve discovers these files automatically. Model provider authentication (the Vercel AI Gateway or an `OPENAI_API_KEY`) follows the [eve quickstart](https://eve.dev/docs/getting-started).

### Run your agent

Start your agent and send it a message, for example _"What's the weather in Berlin? Use the getWeather tool, then answer in one sentence."_:

```bash
npm run dev
```

Each run is now exported to Langfuse.

### View traces in Langfuse

Open your [Langfuse dashboard](https://cloud.langfuse.com) to see the trace for each run. With the `gen_ai`-only filter above, eve's spans map to the Langfuse data model as follows:

- **Agent invocations** become **agent observations**.
- **Model calls** become **generations** with the model name, token usage, and cost.
- **Tool calls** become **tool observations** with their inputs and outputs.

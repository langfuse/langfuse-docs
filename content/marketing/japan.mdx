# Langfuse Cloud Japan

LLM observability, prompt management, evaluation, and metrics — hosted in Japan.

Langfuse Cloud Japan is the managed Langfuse Cloud environment for teams building LLM applications in Japan. It runs on ClickHouse and AWS infrastructure in the AWS Asia Pacific (Tokyo) Region (`ap-northeast-1`).

## Built for LLM teams in Japan

Langfuse is an open-source LLM engineering platform that helps teams debug, analyze, and improve LLM applications in production.

Langfuse Cloud Japan brings the full Langfuse Cloud experience to Japan with local data residency.

Use Langfuse Cloud Japan to:

- Trace LLM applications, agents, retrieval pipelines, and tool usage
- Analyze quality, latency, and cost
- Manage prompts with versioning and releases
- Run evaluations and experiments
- Monitor metrics and production performance

## Why Langfuse Cloud Japan?

### 🇯🇵 Data residency in Japan

All primary application data (traces, prompts, evaluations) is stored in Japan in AWS `ap-northeast-1` using ClickHouse.

### ⚡ Full Langfuse platform

No feature differences — Langfuse Cloud Japan provides the same platform used globally.

### 🏢 Enterprise-ready

- SOC 2 Type II
- ISO 27001
- GDPR-aligned controls
- Data Processing Agreement available

See [Security & Compliance](/security) for details.

## Get started

<Steps>

### Sign up

Create an account at [jp.cloud.langfuse.com](https://jp.cloud.langfuse.com):

1. Create an organization
2. Create a project
3. Generate API keys

### Configure your application

```bash
LANGFUSE_PUBLIC_KEY="pk-lf-..."
LANGFUSE_SECRET_KEY="sk-lf-..."
LANGFUSE_BASE_URL="https://jp.cloud.langfuse.com"
```

### Add tracing

Use an AI coding agent with the [Langfuse skill](https://github.com/langfuse/skills) or [llms.txt](/llms.txt) context.

Example prompt:

```text
Add Langfuse tracing to this app.
Use LANGFUSE_BASE_URL=https://jp.cloud.langfuse.com
```

</Steps>

## Verify Japan region usage

You are using Langfuse Cloud Japan if:

- Your browser URL is `jp.cloud.langfuse.com`
- Your SDK uses:

```bash
LANGFUSE_BASE_URL="https://jp.cloud.langfuse.com"
```

## FAQ

### Where is my data stored?

Primary application data is stored in Japan (AWS `ap-northeast-1`) on AWS and ClickHouse infrastructure.

### Which subprocessors are used?

**Primary subprocessors**

| Subprocessor | Purpose        | Data location |
| ------------ | -------------- | ------------- |
| AWS          | Infrastructure | Japan         |
| ClickHouse   | Database       | Japan         |

**Other subprocessors**

| Subprocessor | Purpose            | Data location |
| ------------ | ------------------ | ------------- |
| Datadog      | Monitoring         | EU            |
| Sentry       | Error tracking     | EU            |
| Google Cloud | Operations         | EU            |
| PostHog      | Product analytics  | EU            |

Full list at [Subprocessors](/security/subprocessors).

### How do I ensure I am using the Japan region?

Check that:

- Your browser URL is `jp.cloud.langfuse.com`
- Your SDK uses:

```bash
LANGFUSE_BASE_URL="https://jp.cloud.langfuse.com"
```

### How does migration work?

See the [data migration guide](/guides/cookbook/example_data_migration). Create a new project in Langfuse Cloud Japan and migrate data from your existing setup.

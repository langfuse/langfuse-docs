# Deprecations

This page tracks every deprecated Langfuse surface, its replacement, and when it is removed. Deprecated features keep working until the removal date (Langfuse Cloud) or removal version (self-hosted), but we recommend migrating as soon as possible.

**How removals work:**

- **Langfuse Cloud:** breaking removals are announced ahead of time with a migration window and happen on a published date.
- **Self-hosted:** breaking removals only happen in major server releases. Within a major version, all updates are non-disruptive.

<Callout type="warning">
**November 30, 2026** is the migration deadline for the [Langfuse v4](/docs/v4) transition on Langfuse Cloud. On this date, all remaining legacy surfaces below are removed, and remaining legacy exports are switched to their replacements automatically.
</Callout>

Notes:

- Score and dataset-run-item events via `POST /api/public/ingestion` are **not** deprecated and keep working on all versions.
- On Cloud, projects created on or after 2026-05-20 cannot select legacy export sources, and new legacy export integrations cannot be created since 2026-06-22.

## Server version lifecycle [#lifecycle]

| Server version | Status      | Support                                                                           |
| -------------- | ----------- | --------------------------------------------------------------------------------- |
| v4             | Preview     | In preview on Langfuse Cloud; launches for self-hosted deployments by Aug 1, 2026 |
| v3             | GA          | Full support; after v4's release, security patches                                |
| v2 and older   | End of life | No support, no security patches                                                   |

SDK majors follow the same pattern: the latest major of each language (Python v4, JS/TS v5) is GA, the previous major is deprecated (works, upgrade recommended), and older majors are end of life. See the full [feature availability matrix](/docs/compatibility#sdk-server).

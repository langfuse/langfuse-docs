# Trace URLs

Each trace has a unique URL that you can use to share it with others or to access it directly.

## Get trace url

Sometimes, it is useful to get the trace URL directly in the SDK. E.g. to add it to your logs or interactively look at it when running experiments in notebooks.

<Tabs items={["Python SDK (v3)", "Python SDK (v2)", "JS/TS", "OpenTelemetry", "Langchain (Python v2)", "Langchain (JS/TS)"]}>
<Tab>
When using the `@observe()` decorator:

```python
from langfuse import observe, get_client

@observe()
def process_data():
    langfuse = get_client()

    # Get the URL of the current trace
    trace_url = langfuse.get_trace_url()
    print(f"View trace at: {trace_url}")

    # or pass the trace id
    trace_id = langfuse.get_current_trace_id()
    trace_url = langfuse.get_trace_url(trace_id=trace_id)
```

When using context managers:

```python
from langfuse import get_client

langfuse = get_client()

with langfuse.start_as_current_span(name="process-request") as span:
    # Get the URL of this trace
    trace_url = langfuse.get_trace_url()
    print(f"View trace at: {trace_url}")

    # or pass the trace id
    trace_id = langfuse.get_current_trace_id()
    trace_url = langfuse.get_trace_url(trace_id=trace_id)
```

When using [OpenTelemetry](/docs/opentelemetry/get-started), you can access trace URLs by first obtaining the trace ID from the OpenTelemetry context and then constructing the URL:

```python
from opentelemetry import trace
import os

tracer = trace.get_tracer(__name__)

with tracer.start_as_current_span("my-operation") as span:
    # Get the trace ID from the span context
    trace_id = format(span.get_span_context().trace_id, "032x")
    
    # Construct the Langfuse trace URL
    # Replace with your actual Langfuse host and project ID
    langfuse_host = os.environ.get("LANGFUSE_HOST", "https://cloud.langfuse.com")
    project_id = "your-project-id"  # Get this from your Langfuse project settings
    
    trace_url = f"{langfuse_host}/project/{project_id}/traces/{trace_id}"
    print(f"View trace at: {trace_url}")
```

Note: When using OpenTelemetry, you'll need to manually construct the trace URL since OpenTelemetry doesn't have a built-in method to get Langfuse-specific URLs. The trace ID format should match the OpenTelemetry trace ID format (32-character hex string).

Learn more about OpenTelemetry integration in the [OpenTelemetry documentation](/docs/opentelemetry/get-started).

Use the interoperability of the Langfuse Python `@observe()` Decorator with the Langchain integration to get the URL of a trace ([interop docs](/docs/integrations/langchain/tracing#interoperability)).

```python
from langfuse.decorators import langfuse_context, observe

@observe()
def fn():
    langfuse_handler = langfuse_context.get_current_langchain_handler()

    # Your Langchain code

    # Add Langfuse handler as callback (classic and LCEL)
    chain.invoke({"input": "<user_input>"}, config={"callbacks": [langfuse_handler]})

    langfuse_context.get_current_trace_url()

fn()
```

**Deprecated:** flaky in cases of concurrent requests as it depends on the state of the handler.

```
handler.get_trace_url()
```

Use the interoperability of the Langfuse SDK with the Langchain integration to get the URL of a trace ([interop docs](/docs/integrations/langchain/tracing#interoperability)).

```ts
// Initialize Langfuse Client
import { CallbackHandler, Langfuse } from "langfuse-langchain";
const langfuse = new Langfuse();

// Create a Langfuse trace for an execution of your application
const trace = langfuse.trace();

// Get Langchain handler for this trace
const langfuseHandler = new CallbackHandler({ root: trace });

// Get the trace URL
langfuseHandler.getTraceUrl();
```

**Deprecated:** flaky in cases of concurrent requests as it depends on the state of the handler.

```ts
handler.getTraceUrl();
```

## Share trace via url

By default, only members of your Langfuse project can view a trace.

You can make a trace `public` to share it via a public link. This allows others to view the trace without needing to log in or be members of your Langfuse project.

_Example: https://cloud.langfuse.com/project/clkpwwm0m000gmm094odg11gi/traces/2d6b96f2-0a4d-4366-99a5-1ad558c66e99_

When using the `@observe()` decorator:

```python
from langfuse import observe, get_client

@observe()
def process_data():
    langfuse = get_client()
    
    # Make the current trace public
    langfuse.update_current_trace(public=True)
```

When using context managers:

```python
from langfuse import get_client

langfuse = get_client()

with langfuse.start_as_current_span(name="process-request") as span:
    # Make this trace public
    span.update_trace(public=True)

    # Get the URL to share
    trace_id = langfuse.get_current_trace_id()
    trace_url = langfuse.get_trace_url(trace_id=trace_id)
    print(f"Share this trace at: {trace_url}")
```

</Tab>

When using [OpenTelemetry](/docs/opentelemetry/get-started), you can make a trace public using the `langfuse.public` span attribute:

```python
from opentelemetry import trace

tracer = trace.get_tracer(__name__)

with tracer.start_as_current_span("my-operation") as span:
    # Make the trace public using the Langfuse-specific attribute
    span.set_attribute("langfuse.public", True)
```

You can also set traces to be public globally using resource attributes:

```python
import os

# Set all traces to be public by default
os.environ["OTEL_RESOURCE_ATTRIBUTES"] = "langfuse.public=true"
```

**Supported OpenTelemetry attributes for making traces public:**

- `langfuse.public` - Boolean value to make the trace publicly accessible (only applied to root spans)

Note: The `langfuse.public` attribute is only considered when set on the root span of a trace, as the public status is a trace-level property in Langfuse.

Learn more about OpenTelemetry integration in the [OpenTelemetry documentation](/docs/opentelemetry/get-started).

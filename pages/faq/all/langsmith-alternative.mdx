# Langfuse vs. LangSmith

This guide outlines the key differences between **[Langfuse](/)** and **[LangSmith](https://www.langchain.com/langsmith)** to help engineering teams choose the right LLM observability platform.

**TL;DR:**

* **Choose Langfuse** if you prioritize **Open Source** (MIT), data sovereignty (full self-hosting), a framework-agnostic approach (works with any stack), and transparent unit-based pricing.
* **Choose LangSmith** if you are an **"All-in-LangChain"** shop requiring a managed SaaS solution that offers deep, native integration for deploying LangGraph agents ("DevOps-in-a-box").

## Open Source & Distribution

Langfuse operates on an "Open Core" model where the platform is open source (MIT) and self-hosting is a first-class citizen. LangSmith is a proprietary, closed-source SaaS tool; while it offers a self-hosted option, it is gated strictly behind an Enterprise license.

| Feature | Langfuse | LangSmith |
| --- | --- | --- |
| **Model** | **[Open Source](https://github.com/langfuse/langfuse)** (MIT License) | **Proprietary SaaS** (Closed Source Backend) |
| **GitHub Stars** | [](https://github.com/langfuse/langfuse) | Nhttps://www.google.com/search?q=/A (Server source code is not public) |
| **Self-Hosting** | **First-Class Citizen**: Full feature parity with Cloud. Free to deploy via [Docker/Helm](https://www.google.com/search?q=/self-hosting). | **Enterprise Only**: Requires a sales contract and license key. |
| **Data Sovereignty** | **High**: Can run fully air-gapped on own VPC without vendor contact. | **Medium**: "Hybrid" deployment available for Enterprise; SaaS defaults to Cloud. |

## Scalability & Performance

Both platforms utilize ClickHouse for high-scale analytics. However, Langfuse's architecture is natively built on ClickHouse (and recently acquired by ClickHouse Inc.), whereas LangSmith abstracts this infrastructure behind a managed SaaS layer.

| Feature | Langfuse | LangSmith |
| --- | --- | --- |
| **Backend** | **[ClickHouse](https://clickhouse.com/)**: Native OLAP architecture for high-throughput observability. | **ClickHouse + Postgres**: Enterprise-scale architecture managed by LangChain. |
| **Ingestion** | **Async Queue**: Decoupled Redis queue + generic workers for reliability. | **Queue-Based**: Stateless backend designed for horizontal scaling. |

## Pricing

The economic models differ significantly. Langfuse charges based on the *depth* of data (Units), while LangSmith charges based on the *volume of root executions* (Traces), largely ignoring internal steps.

| Feature | Langfuse | LangSmith |
| --- | --- | --- |
| **Model** | **[Unit-Based](https://www.google.com/search?q=/docs/administration/billable-units)**: 1 Unit = 1 Trace, Observation, or Score. | **Trace-Based**: Charges per root run (internal steps included). |
| **Best For** | **Cost-Efficiency**: Cheaper for high-volume, shallow chains (e.g., RAG chatbots). | **Deep Agents**: Cheaper for highly recursive, complex agents (e.g., 50+ step autonomous loops). |
| **Free Tier** | **50,000 unitshttps://www.google.com/search?q=/mo** (highly generous for developers). | **5,000 traceshttps://www.google.com/search?q=/mo**. |
| **Plans** | [Free, Core ($29/mo), Pro ($199/mo), Enterprise](https://www.google.com/search?q=/pricing). | [Developer (Free), Plus ($39/seat/mo), Enterprise](https://www.langchain.com/pricing). |

## Open Platform & Extensibility

Langfuse is API-first, allowing teams to treat observability data as their own. LangSmith focuses on extending the LangChain ecosystem via the Hub.

| Feature | Langfuse | LangSmith |
| --- | --- | --- |
| **API Access** | **Full CRUD**: Public API for Traces, Spans, Scores, and Prompts. | **Readhttps://www.google.com/search?q=/Write**: API available for runs and datasets. |
| **Data Export** | **[Blob Storage Export](https://www.google.com/search?q=/docs/api-and-data-platform/features/export-to-blob-storage)**: Automated dumps to S3https://www.google.com/search?q=/GCP (JSONLhttps://www.google.com/search?q=/Parquet). | **Bulk Export**: Available on paid plans. |
| **Ecosystem** | **Neutral**: Integrates with any LLM stack or gateway. | **LangChain Hub**: Native integration for prompt sharing. |

## Enterprise Security

Both platforms are enterprise-ready with major certifications. Langfuse offers ISO 27001 in addition to SOC 2, and allows for easier air-gapped compliance via open-source self-hosting.

| Feature | Langfuse | LangSmith |
| --- | --- | --- |
| **Certifications** | **[SOC 2 Type II, ISO 27001](https://www.google.com/search?q=/security)**, GDPR, HIPAA aligned. | [SOC 2 Type II, GDPR, HIPAA](https://docs.langchain.com/langsmith/pricing-faq). |

## Feature Highlights

**Langfuse:**

* **[Model Agnosticism](https://www.google.com/search?q=/docs/integrations/overview):** Works equally well with OpenAI SDK, LiteLLM, LlamaIndex, or raw HTTP calls.
* **[Prompt Management](https://www.google.com/search?q=/docs/prompt-management/overview):** Agnostic playground that doesn't lock you into a specific framework's syntax.
* **[Evaluations](https://www.google.com/search?q=/docs/evaluation/overview):** "LLM-as-a-judge" evaluators that run on your own infrastructure or via the managed service.
* **[ClickHouse Native](https://www.google.com/search?q=/blog/2024-12-langfuse-v3-infrastructure-evolution):** Unrestricted access to raw data via SQL (in self-hosted) for custom analytics.

**LangSmith:**

* **[LangGraph Deployment](https://www.langchain.com/langsmith/deployment):** A specialized runtime to deploy LangGraph agents as APIs (DevOps capabilities).
* **Zero-Setup Tracing:** Automatic instrumentation for LangChain applications via environment variables.
* **[The Hub](https://www.google.com/search?q=https://smith.langchain.com/hub):** Access to a community repository of prompts.
